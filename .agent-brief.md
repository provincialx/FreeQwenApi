# FreeQwenApi Proxy — Agent Brief (Session 2026-06-07)

## Архитектура
FreeQwenApi = браузерный прокси ForgetMeAI. Node.js + Puppeteer.
*   Заменяет веб-аккаунт Qwen Chat на локальный OpenAI-compatible API (`localhost:3264`)
*   Эмулирует вызовы Qwen через JS `fetch` внутри puppeteer-страницы
*   Zed/Aider -> FreeQwenApi -> Qwen Chat (через браузер) -> ответ обратно

## Найденные проблемы
1. **`resolveQwenChatId` без return** → функция возвращала `undefined`, маппинг чатов не сохранялся, каждый запрос создавал новый Qwen-чат.
2. **Орфан-импорт посреди файла** (`import { testToken }`) → нарушало структуру JS модуля.
3. **"Потрошное месиво" в логах** (273 сообщения `user, assistant, tool...`) → бесконечная строка ролей забивала консоль и убивала контекст при agent-loop (tool_calls).
4. **Смерть сессий (`chat_not_exist`)** → Qwen удалял старые чаты, но проши продолжал использовать их кэшированный `defaultForModel.chatId`. Результат: 500 Bad_Request.
5. **Неряшливый лог ошибок** (`undefined`) → `handleApiError` возвращал `{ error: undefined }`, что routes.js считал успехом (falsy check). Клиент получал пустой ответ вместо текста ошибки.

## Применённые патчи
| # | Фикс | Файл | Суть |
|---|------|------|------|
| 1 | `resolveQwenChatId` return + антиспам чатов | `routes.js` | Добавлен финальный `return qwenChatId`. Блок создания нового чата теперь срабатывает только если нет ни маппинга, ни дефолтного. Импорт `testToken` перемещён в хедер. |
| 2 | Graceful ошибки + errorBody WARN | `chat.js` | Финальный return в `handleApiError`: всегда строка (`HTTP 500`, `Неизвестная ошибка` и т.д.). Логирование тела ответа поднято с DEBUG до WARN (видимо без переключения уровня). Добавлен fallback для неполных объектов ошибки. |
| 3 | Auto-retry при "chat not exist" | `routes.js + chat.js` | При получении от Qwen: `"is not exist"` → инвалидировать кэш модели, создать свежий чат через `createChatV2`, повторить запрос ровно один раз (retryCount === 0). Добавлены хелперы: `invalidateModelDefaultChat()`, `isChatNotExistError()` |
| 4 | Sliding Window + компактные логи ролей | `routes.js` | Если история >60 сообщений, срезается до System Prompt + последние 40. Логирование бесконечной строки ролей заменено на сводку: `273 сообщений (1x system, 85x user...)`. Спасает от гигантских JSON-payload и падений Puppeteer. |
