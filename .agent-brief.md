# FreeQwenApi Proxy — Agent Brief (Session 2026-06-07)

## Архитектура
FreeQwenApi = браузерный прокси ForgetMeAI. Node.js + Puppeteer.
*   Заменяет веб-аккаунт Qwen Chat на локальный OpenAI-compatible API (`localhost:3264`)
*   Эмулирует вызовы Qwen через JS `fetch` внутри puppeteer-страницы
*   Zed Agent (Browser) → FreeQwenApi → Qwen Chat (через браузер) → ответ обратно
*   Цель: Полноценный агент с tool_calls через интерфейс Zed. Терминальные обёртки исключены.

## Zed.dev + chat.qwen.ai (что узнали)
### Zed
- Rust-based редактор, GPU-accelerated GPUI
- Agentic Editing: встроенные AI-агенты с параллельным выполнением, MCP-серверы
- Edit Prediction через Zeta2 модель. Vim-friendly modal editing
- Any Agent, Any Tool — поддерживает ACP (Agent Communication Protocol), Claude/Codex/OpenCode agents
- Подключается к OpenAI-compatible API: `base_url`, `api_key` в settings

### Qwen Studio (chat.qwen.ai)
- Веб-чат Qwen. Mobile-first, требует скачивания App на десктопе ("Current System does not Support")
- Для прокси не важно — мы стучимся к `/api/v2/chat/completions` через Puppeteer, а не к фронтенду

## Найденные проблемы
### Сессия 1 (решены)
1. **`resolveQwenChatId` без return** → функция возвращала `undefined`, маппинг чатов не сохранялся.
2. **Орфан-импорт посреди файла** (`import { testToken }`) → нарушало структуру JS модуля.
3. **«Потрошное месиво» в логах** (273 сообщения) → бесконечная строка ролей забивала консоль.
4. **Смерть сессий (`chat_not_exist`)** → Qwen удалял старые чаты, прокси использовал кэшированный ID.
5. **Неряшливый лог ошибок** (`undefined`) → `handleApiError` возвращал falsy `{ error: undefined }`.

### Сессия 2 (решены сегодня)
6. **Отсутствие `protocolTimeout` в puppeteer.launch()** → дефолт 180с убивает большие задачи.
7. **Бесконечный hang health-check пула страниц** → `page.evaluate()` без таймаута.
8. **Нет retry при сетевых ошибках goto** → мгновенный фейл на `ERR_CONNECTION_RESET`.
9. **Sliding window рвёт tool_call пары** → `.slice(-40)` мог оставить orphan tool без assistant context.

### Сессия 3 (решены сегодня)
10. **SyntaxError L1287 — незакрытый object literal** → Node ронялся при старте.
11. **applyToolPrompt не принимал `inAgentLoop`** → agent-loop с tool results получал агрессивный prompt вместо light mode. Фикс: `toolsToLightPrompt()` + 3-й параметр.

### Сессия 4 (решены сегодня)
12. **Warn "модель игнорирует инструменты" не работал** → check стоял в chat.js по параметру `tools` который всегда `null`.

### Сессия 5 (решено сегодня)
13. **Портянка километровая в логах** → `logRaw()` дампил полные JSON-тела, debug-trace middleware спамил на каждый запрос к `/chat*`, LOG_LEVEL дефолт = `'debug'`. Всё убрано.

## Критический код — как работает сегодня
```
Zed Agent POST /api/chat/completions { messages: [...], tools: [...] }
  → routes.js парсит body, извлекает system message + combinedTools
    → applyToolPrompt(systemMessage, combinedTools, inAgentLoop) добавляет инструмент-схемы к системному промпту (не к Qwen API!)
      → prepareOpenAIMessageInput() сворачивает OpenAI transcript в единое user-сообщение для Qwen Chat
        → resolveQwenChatId(effectiveChatId, model) → маппинг generated→qwen ID. Если не существует — auto-retry с новым чатом.
          → sendMessage(messageContent, model, qwenChatId, ...) через Puppeteer fetch к /api/v2/chat/completions
            → executeApiRequest() → Qwen отвечает SSE stream (или JSON fallback)
              → если tool_calls в ответе: parseToolCallJson(content) → OpenAI message.tool_calls format
                → routes.js возвращает клиенту через buildOpenAIToolResponse() или writeToolCallsSse() (streaming)
```
